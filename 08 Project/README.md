# 최종 Project 알아보기 

## 프로젝트 목표
- 기업 수준의 AI Agent 기반 서비스/플랫폼 개발 (E2E: 파싱→ETL→DB→Agent→평가→배포)

## 주제 선정 (팀별 택 1)

각 팀은 아래의 시나리오 중 하나를 선정하여 비즈니스 전략을 수립하고 WBS를 작성한다.

- `계약서 분석 Agent` : 복잡한 법률 및 계약 조항 추출 및 위반 사항 자동 검토
- `OCR 기반 고객 문서 자동 처리 Agent` : 청구서, 신청서 등 다양한 고객 제출 서류의 자동 분류 및 데이터화
- `Meeting → Task 자동 생성 Agent` : 회의록 분석을 통한 핵심 안건 도출 및 담당자별 업무 할당 자동화

## 필수 프로젝트 기능 구현

- `데이터셋 확보 및 수집`: AI Hub, 공공데이터, Kaggle 또는 합성 데이터(Synthetic Data) 생성 및 수집
- `데이터 전처리 및 ETL 자동화`: 비정형 데이터 파싱, 중복 제거, 개인정보 비식별화 및 Airflow 기반 적재 자동화
- `지식 DB 및 검색 엔진 구축`: Hybrid RAG(Vector+Graph) 구현 및 의미론적 검색 최적화
- `멀티 에이전트 로직 구현`: Planner-Executor 구조 기반의 자율 해결 에이전트 및 도구 라우팅 구현

## 프로젝트 문서화

- 프로젝트 추진 전략(WBS)
- 데이터 파이프라인 설계도
- 하이브리드 RAG 구축 결과
- 에이전트 답변 성능
- (LLM-as-a-Judge) 및 검증 결과 확인
